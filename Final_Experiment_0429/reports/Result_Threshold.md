# 실험 결과 — Y Threshold Sweep (T = 5% / 10% / 15% / 20%)

ICE 도메인 7,086 patents 에 대해 **Y 라벨링 threshold 변화에 따른 분류 성능
변화** 분석. 핵심 질문:

> Debate 변수의 추가 효과가 threshold 에 따라 어떻게 달라지는가?
> 어떤 T 에서 debate 가 가장 빛나는가?
> 모델별로 어떻게 다른가?

기본 분석 (`Result.md`) 은 T=10% (q90) 만 다룸. 이번엔 T ∈ {5, 10, 15, 20}%
4 가지 threshold × **6 가지 모델** (RF / GBT / XGB / LogReg / SVM / FFN) 에서
동일 protocol 재실행 후 비교.

---

## 실험 설정

| 항목 | 값 |
|---|---|
| Universe | 7,086 USPTO patents (Sinigaglia 2022 16-query union, 1980–2020) |
| Threshold sweep | T ∈ {5, 10, 15, 20}% |
| Y_T 정의 | `forward5 >= percentile(forward5, 100−T)` |
| Train / Test split | 80 / 20 stratified random on Y_T |
| Protocol | 20 seeds (42, 0..18) mean ± std per T |
| 전처리 | median impute + min-max scale (train fold만 fit) |
| Test n | 1,418 |
| 총 모델 fit | 4 T × 6 모델 × 3 configs × 20 seeds = **1,440 fits** |

### 6 모델

| Model | 종류 | Hyperparameters |
|---|---|---|
| **RF** | Random Forest (bagging) | n_estimators=300, n_jobs=-1 |
| **GBT** | Gradient Boosting (sklearn) | n_estimators=300, max_depth=3 |
| **XGB** | XGBoost (boosting + regularization) | n_estimators=300, tree_method=hist |
| **LogReg** | Logistic Regression (linear) | solver=lbfgs, max_iter=1000 |
| **SVM** | Support Vector Machine | kernel=rbf, probability=True |
| **FFN** | Feedforward Neural Net (MLP) | hidden=ceil(sqrt(n_features × 2)), logistic, lbfgs |

3 가지 feature configuration (이전 실험과 동일):
- **BIBLIO** — 26 features (15 numeric biblio + MF top-10 one-hot + MF_other)
- **+25** — 51 features (BIBLIO + 25 debate 변수 전체)
- **+6focal** — 32 features (BIBLIO + 6 focal: var_conf_pro, H_final, delta_H, conf_gap_change, cross_domain_attack, semantic_coherence)

성능 지표 (top-T% prediction threshold 기준): AUROC, AUPRC, Precision, Recall, F1, DOR (Haldane).

## 핵심 관찰

### 1. **T=5% 에서 debate 가 폭발적 효과** — 통계적 매우 유의

**XGB +6focal / +25 가 19/20 ~ 18/20 seeds 에서 BIBLIO 를 이김** (binomial p < 0.001).
Δ AUROC = +0.032 ~ +0.034 (T=10% 의 2.4-9× 크기).

→ **진짜 breakthrough patent (top 5%) 식별엔 debate 가 결정적 도움**.
→ multi-class 분석에서 본 "L1 (top 2%) AUROC 0.83 으로 가장 잘 분류됨" 결론과 일치.

### 2. **T=10% 가 debate 효과 dip 구간**

흥미롭게도 T=10% 가 우리가 default 로 사용해온 threshold 인데,
threshold sweep 관점에서는 **debate uplift 가 가장 작은 구간**:

| Model | T=5% Δ | T=10% Δ | T=15% Δ | T=20% Δ |
|---|---:|---:|---:|---:|
| XGB +6focal | +0.032 | **+0.014** ↓ | +0.015 | +0.012 |
| SVM +6focal | +0.027 | **+0.011** ↓ | +0.019 | +0.017 |
| LogReg +6focal | +0.007 | +0.003 ↓ | +0.007 | +0.007 |

가설: T=10% 가 자연스러운 q90 boundary (forward5 = 6) 인데, 이 boundary 근처
patent 들은 biblio 만으로도 잘 식별됨 (TCT, PK 같은 biblio 변수 가 q90 근처에서
sharp 분리). T=5% (더 극단) 또는 T=15-20% (더 boundary 모호) 일 때 debate 가
추가 정보 제공.

### 3. **모델별 debate 활용 능력 hierarchy** (모든 T 에서 일관)

| 순위 | Model | 평균 +6focal Δ AUROC across T | 특징 |
|:---:|---|---:|---|
| 1 | **XGB** | **+0.018** | debate 신호 추출 최고; T=5% 에서 압도적 (+0.032) |
| 2 | SVM | +0.017 | 일관된 활용; T=5% 와 T=15-20% 에서 강함 |
| 3 | LogReg | +0.006 | 작지만 매우 일관 (T=5/15/20% 각 17/20 wins) |
| 4 | GBT | +0.003 | 거의 미미 — XGB 와 architecture 비슷하지만 차이 |
| 5 | RF | +0.004 | 거의 미미 |
| 6 | FFN | −0.016 | 항상 손해 — 51 features 에서 과적합 |

### 4. **GBT vs XGB — 중요한 발견**

GBT 와 XGB 는 둘 다 gradient boosting 인데 debate 활용도가 매우 다름:

| | GBT | XGB |
|---|---:|---:|
| BIBLIO AUROC (T=5%) | 0.759 | 0.709 |
| +6focal Δ (T=5%) | +0.008 | **+0.032** |
| +25 Δ (T=5%) | +0.009 | **+0.034** |
| +6focal wins (T=5%) | 13/20 | **19/20** |

→ **XGB 는 BIBLIO 에서 약하지만 debate 추가 시 점프**.
→ **GBT 는 BIBLIO 에서 더 강하지만 debate 활용 미미**.

원인 가설:
- XGB 의 L1/L2 regularization 이 noisy debate features 를 robustly 다룸
- GBT 는 stage-wise greedy 로 debate features 의 marginal info 캐치 못함
- `tree_method=hist` 의 효율적 split finding 이 debate 변수의 nonlinear interaction 발견

→ **architecture detail 이 debate 활용도에 큰 영향** — 단순히 "boosting" 카테고리로 묶을 수 없음.

### 5. **LogReg — 선형 모델의 일관성**

LogReg 는 절대 AUROC 는 중간 (RF/GBT 보다 약, XGB 보다 강) 이지만
**debate uplift 의 win rate 가 높음** (T=5/15/20% 각 17/20 wins, T=10% 13/20):

| T | LogReg +6focal Δ | wins | XGB +6focal Δ | wins |
|---|---:|:---:|---:|:---:|
| 5% | +0.007 | 17/20 | +0.032 | 19/20 |
| 10% | +0.003 | 13/20 | +0.014 | 17/20 |
| 15% | +0.007 | 17/20 | +0.015 | 17/20 |
| 20% | +0.007 | 17/20 | +0.012 | 18/20 |

→ Linear model 도 debate features 를 활용 가능. Win rate 일관 → 신뢰성 높음.
→ Magnitude 는 작지만 robust — interpretability + debate 신호 활용을 동시에 원할 때 LogReg 도 옵션.

### 6. **+25 vs +6focal — T 의존성**

| T | XGB +6 | XGB +25 | 차이 |
|---|---:|---:|---:|
| **5%** | +0.032 | **+0.034** | **+25 약간 우세** |
| 10% | +0.014 | +0.004 | +6focal 우세 |
| 15% | +0.015 | +0.013 | +6focal 우세 |
| 20% | +0.012 | +0.009 | +6focal 우세 |

→ **T=5% 에서는 25개 전체 debate 변수가 가치 있음**.
→ 다른 T 에서는 +6focal 우세 — sparse 가 better.
→ Strict threshold 에서는 모든 debate 신호가 결정적, 다른 T 에서는 6 focal 만으로 충분.

### 7. **FFN +25 절대적 패배** — 모든 T 에서 0/20 또는 거의 0 wins

T=10/15/20% 에서 +25 / FFN: **0/20 wins**.
T=5% 에서도 2/20.

→ 51-feature × ~5,668 train sample 에서 MLP lbfgs 과적합 명백.
→ 본 데이터 규모에서 **augmented 모델 시 FFN 사용 금지** (이전 결론 재확인).

## 결론

| 질문 | 답 |
|---|---|
| Threshold 별 debate 효과 차이? | **Yes, dramatically**. T=5% 에서 가장 크고 T=10% 에서 dip. |
| 어떤 T 에서 debate 가 빛나는가? | **T=5% (top breakthrough)**. XGB +6focal Δ +0.032, 19/20 wins. |
| 어떤 모델이 debate 활용 best? | **XGB 1위, SVM 2위, LogReg 3위**. RF/GBT 거의 미미. FFN 항상 손해. |
| GBT 도 XGB 처럼 debate 활용? | **No** — 같은 boosting family 인데 GBT 는 거의 활용 못함 (architecture 차이 큼). |
| LogReg (linear baseline) 의미는? | 작지만 매우 일관된 debate uplift (T=5/15/20% 각 17/20 wins). interpretability 우선시 시 옵션. |
| 절대 AUROC 1위 모델? | **RF** (모든 T 에서 1위). |
| 새로운 통찰? | **이전에 "debate 효과 modest"로 결론 내린 것은 T=10% 만 봤기 때문**. T=5% 에서 보면 substantial (Δ +0.032). |

### Final 추천

| 사용 케이스 | 권장 모델 + config |
|---|---|
| Top breakthrough (T=5%) 식별 | **XGB + +6focal** (Δ +0.032) 또는 +25 (Δ +0.034) |
| 일반 deployment (T=10%) | **RF + 6focal** (절대 AUROC 1위) |
| Loose threshold (T=15-20%) | RF + 6focal (절대) 또는 SVM + 6focal (debate Δ 큼) |
| Interpretability 필요 | **LogReg + 6focal** (일관 +0.007 lift, 17/20 wins, 선형) |
